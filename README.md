# PicoSmartHub

PicoSmartHubは、Raspberry Pi Pico W 2（およびPico W）をコアとして開発された、多機能かつ統合的なスマートホームハブ兼環境測定ステーションです。
単なるセンサーノードにとどまらず、外部クラウドサービス（SwitchBot, LINE, Google Apps Script）との連携、赤外線通信、および独自のローカルネットワーク内での子機連携を安定して処理できるシステムとして設計されています。

### 電源供給について
本システムは、設置環境に応じて「USB給電」と「乾電池」の2種類の電源方式で動作するように設計されています。これにより、コンセントがない場所でもスタンドアローンで稼働させることが可能です。
**USBケーブルによる給電と、乾電池による給電を絶対に同時に行わないでください。**

## コア機能と技術詳細

本システムは、日常生活の利便性向上と環境監視を目的として、以下の主要な機能を備えています。

### 1. 高度な環境センシングと異常検知
本体および外部センサーを用いて、生活環境を常に監視します。
- **高精度な温湿度測定**: 本体内蔵のDHT20で室内の温湿度を常時測定します。また、屋外や別室を測定するための外部センサー（DHT22）は、本体のイヤホンジャック端子を利用して接続する設計となっており、用途に合わせて容易に着脱が可能です。すべての測定結果はLCDに一覧表示されます。
- **雷雲接近のリアルタイム検知**: AS3935雷センサーモジュールを搭載。ハードウェア割り込み（IRQ）を用いてバックグラウンドで雷の発生を監視し、検知時には「雷までの推定距離」を即座に算出して画面に警告表示します。同時にRGB LEDを黄色点滅させ、視覚的にも危険を知らせます。

### 2. 子機（PicoSmartHub_Sub）とのローカルネットワーク連携
同一Wi-Fiネットワーク内に配置した別プロジェクトの子機（[PicoSmartHub_Sub](https://github.com/kazu-1234/PicoSmartHub_Sub)）とTCP/IP経由で直接通信を行い、リアルタイムな状態共有を実現しています。
- **お風呂の湯張り監視システム**: 子機に搭載された超音波距離センサーが浴槽の水位を監視します。水位が事前に設定した閾値に達すると、子機から親機に対してプッシュ通知（HTTPリクエスト等）が送信されます。
- **親機側のアラート処理**: 通知を受け取った親機は、現在実行中の画面表示を強制的にオーバーライドして「お風呂が沸いたこと」を画面に大きく表示し、同時にRGB LEDを白色の無限点滅モードに切り替えてユーザーに確実に知らせます。

### 3. 統合的な家電制御（SwitchBot API & ローカル赤外線）
赤外線方式の家電を本機から一括制御します。
- **SwitchBotクラウド連携**: `config.h` に登録したSwitchBotのAPIトークンおよびデバイスIDを利用し、ネットワーク経由で照明、テレビ、エアコンなどを本体の物理ボタン・メニューから直接操作可能です。APIのレスポンス処理もマイコン内で完結しています。
- **赤外線（IR）の学習と送信**: 赤外線受信モジュールを用いて既存のリモコンの信号を読み取り、マイコン内に記録する学習モードを実装。記録した信号は赤外線送信LEDから出力でき、ローカル環境のみで完結する遅延のない家電操作を実現しています。

### 4. 双方向の外部サービス連携（GAS & LINE API）
マイコン単体では難しいデータの蓄積や遠隔からの操作を、Google Apps Script (GAS) とLINE Messaging APIを組み合わせることで解決しています。
- **環境データのクラウドロギング**: 定期的に温湿度データや雷の検知ログをGASへ送信し、Googleスプレッドシートに自動追記します。これにより、過去の環境データのグラフ化や分析が可能です。
<img src="https://github.com/user-attachments/assets/a693e2a2-fbca-4bbc-a938-a0d92fd54f52" alt="LINEからの操作" width="300" />

- **LINEへの自動警告と遠隔操作**: 雷接近時にLINEへ自動でメッセージをプッシュ送信します。また、外出先からLINEのBot宛てに特定のメッセージ（例：「PC起動」）を送信すると、GASを経由して親機がそのリクエストを検知し、後述のWoL機能などを実行する双方向通信を構築しています。

### 5. Wake on LAN (WoL) によるPC電源管理
あらかじめ設定ファイルに登録されたデスクトップPCやホームサーバーのMACアドレスに対し、マジックパケットを生成・ブロードキャスト送信します。これにより、物理的なスイッチを押すことなく、本機からのメニュー操作やLINE経由でのリモート起動が可能です。

### 6. デュアルコアを活用したシステムアーキテクチャ
Raspberry Pi Pico W 2 のRP2350（またはRP2040）のデュアルコアを最大限に活用し、処理のブロッキングを防いでいます。
- **Core 0**: センサーの読み取り、LCDの描画、ボタン入力の判定、赤外線の送受信など、タイミングがシビアなハードウェア制御とUI処理を担当。
- **Core 1**: Wi-Fiの接続維持、HTTPリクエストの送信（SwitchBot/GAS等）、WoLパケットの生成、子機からの通信待受など、時間のかかる重いネットワーク処理をバックグラウンドで担当。

## 回路構成・ピンアサイン詳細

Raspberry Pi Pico W 2 (Pico W 互換) との結線は以下の通りです。

- **LCDディスプレイ (20x4文字表示 / LiquidCrystal互換)**
  - RS: GP2
  - E: GP3
  - D4: GP4, D5: GP5, D6: GP6, D7: GP7
  - バックライト (PWM調光用): GP14
- **RGB LED (ステータス・アラート用)**
  - Red: GP28, Green: GP27, Blue: GP26
- **ユーザーインターフェース**
  - 操作ボタン入力: GP15 (内部プルアップ有効)
- **I2Cバス (各種センサー通信用)**
  - SDA: GP0
  - SCL: GP1
  - 接続デバイス: DHT20 (室内温湿度), AS3935 (雷センサー)
- **センサー個別ピン**
  - AS3935 割り込み (IRQ): GP16
  - DHT22 (屋外用温湿度) データ: GP17 （※イヤホンジャック経由で着脱可能）
- **赤外線 (IR) モジュール**
  - IR受信 (OUT): GP18
  - IR送信LED (IN): GP19

## 導入・セットアップ手順

1. **開発環境の準備**
   Arduino IDEを使用します。ボードマネージャで Raspberry Pi Pico / RP2040 (earlephilhower版) をインストールしてください。
2. **必須ライブラリのインストール**
   ライブラリマネージャから以下のライブラリをインストールします。
   - `LiquidCrystal`
   - `DFRobot_DHT20`
   - `DHT sensor library` (Adafruit)
   - `SparkFun AS3935 Lightning Sensor`
   - `ArduinoJson`
   - `IRremote`
3. **`config.h` の設定**
   プロジェクト内の `config.h` を環境に合わせて編集してください。
   - Wi-Fi接続情報（複数APのフォールバック対応）
   - 固定IP・ゲートウェイ設定
   - 子機（PicoSmartHub_Sub）のIPアドレス
   - WoL対象機器のMACアドレス
   - 各種APIキー（SwitchBot, LINE, GASデプロイURL）
4. **コンパイルと書き込み**
   ボードを「Raspberry Pi Pico W」等に設定し、書き込みを行います。初回起動時はWi-Fi接続とNTPサーバーによる時刻同期が行われ、完了後にメインシステムが立ち上がります。
